Project Title:
World Chat – Real-time Global Chat Application
________________________________________
Objective:
Design and build a modern, real-time chat application using FastAPI (backend) and MongoDB Atlas, with a visually appealing glassmorphism (glass blur water soft) UI. The app must support authentication, guest access, admin controls, and message moderation.
________________________________________
🌊 UI/UX Theme Requirements
🎨 Design Style:
•	Glassmorphism (blurred glass effect)
•	Soft water-like gradients
•	Smooth shadows and transparency
•	Minimal, clean, modern UI
🌞 Light Theme:
•	Background: soft white + light blue gradient
•	Glass cards: semi-transparent white with blur
•	Accent: sky blue / aqua
•	Text: dark gray / black
🌙 Dark Theme:
•	Background: deep navy / black gradient
•	Glass cards: translucent dark glass with blur
•	Accent: neon blue / cyan glow
•	Text: white / light gray
✨ Effects:
•	Backdrop blur (frosted glass)
•	Smooth transitions (0.3s ease)
•	Rounded corners (12px–20px)
•	Subtle glow for active elements
________________________________________
👥 User Types & Roles
1. Guest User:
•	Can join instantly without registration
•	Assigned random username (e.g., Guest_1234)
•	Can send messages
•	Cannot edit/delete messages after sending
2. Registered User:
•	Register using email + password
•	Login using JWT authentication
•	Can send messages
•	Can delete only their own messages
3. Admin User:
•	Special role ("admin")
•	Can delete ANY message
•	Can edit ANY message
•	Can send highlighted/admin messages
________________________________________
🔐 Authentication System
•	Email + Password आधारित registration
•	Password hashing using bcrypt
•	JWT-based login system
•	Protected routes using token validation
•	Store user role (user/admin) in database
________________________________________
💬 Chat System Features
Core Features:
•	Real-time messaging using WebSockets
•	Global chat room (all users)
•	Message persistence using MongoDB
Message Structure:
•	Username
•	Message text
•	Timestamp (date + time)
•	is_admin flag
•	is_edited flag
________________________________________
🧠 Moderation System
•	Implement word filter for abusive/hate words
•	Block or sanitize messages before saving
•	Example:
o	"badword" → "****"
•	Messages should be validated before broadcasting
________________________________________
⚙️ Functional Requirements
Messaging:
•	Users can send messages instantly
•	Messages appear in real-time for all users
Delete Logic:
•	User can delete only their own messages
•	Admin can delete all messages
Edit Logic:
•	Only admin can edit messages
•	Edited messages should show "(edited)" tag
________________________________________
🎨 UI Components
Chat Layout:
•	Header:
o	App name (World Chat)
o	Theme toggle (Light/Dark)
o	User info (Guest/User/Admin)
•	Chat Area:
o	Scrollable message list
o	Each message card:
	Username
	Message
	Timestamp
	Delete icon (conditional)
	Edit icon (admin only)
•	Input Box:
o	Glass-style input field
o	Send button with icon
________________________________________
🧩 Message UI Behavior
•	Own messages aligned right
•	Others' messages aligned left
•	Admin messages highlighted (different color/glow)
•	Smooth fade-in animation on new messages
________________________________________
🛡️ Security & Best Practices
•	Use environment variables for MongoDB URI
•	Validate all inputs
•	Prevent unauthorized actions (delete/edit)
•	Implement basic rate limiting (optional)
________________________________________
🗄️ Database Design (MongoDB)
Users Collection:
•	username
•	email
•	password (hashed)
•	role (user/admin)
Messages Collection:
•	user_id (nullable for guest)
•	username
•	message
•	timestamp
•	is_admin
•	is_edited
________________________________________
🚀 Tech Stack
•	Backend: FastAPI (Python)
•	Database: MongoDB Atlas
•	Real-time: WebSockets
•	Frontend: React.js vite / HTML-CSS-JS (glass UI)

🎯 Extra Enhancements (Optional but Recommended)

•	Auto-scroll to latest message
